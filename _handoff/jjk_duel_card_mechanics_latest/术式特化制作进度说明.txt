咒术回战卡牌对战机制交接包说明
生成日期：2026-04-30

一、包内容范围

本压缩包整理的是当前工作区内“咒术回战”卡牌对战模式的最新机制文件，重点覆盖：

1. 卡牌数值计算机制
   - 卡牌最终预览与执行数值
   - 咒力消耗
   - 伤害
   - 格挡
   - 领域压力
   - 资源恢复与稳定性
   - 手牌候选与行动模板

2. 与基础数值产生关联的算法
   - 角色基础数值到 raw score 的换算
   - raw score 到 axes 的战斗评估
   - axes 到 combatScore / disruptionScore 的综合评分
   - 基础数值到卡牌模板 finalCost / finalDamage / finalBlock / domainPressure 的换算
   - 特殊天赋、机制卡、术式关键词对数值的影响路径

3. 卡牌文案
   - 行动模板
   - 卡牌模板
   - 卡牌文案候选
   - 机制模板
   - 领域、手牌、终局与试炼目标规则

4. 已有角色卡
   - character 目录下的现有角色 JSON
   - character/manifest.json
   - 角色数据说明文本

三、已完成事项

1. 卡牌对战基础数值链路已建立

已完成从角色基础数值到对战评分的链路：

角色基础数值
-> raw score
-> active mechanisms
-> axes
-> combatScore / disruptionScore
-> duelResource
-> card preview / action result

其中角色评估主要由 tool/runtime-fight.js 中的 evaluateDuelCharacterCard(card) 承担。

2. 卡牌模板数值计算已建立

modules/duel/duel-card-template.js 已负责：

- finalCost：最终咒力消耗。
- finalDamage：最终伤害。
- finalBlock：最终格挡。
- domainPressure：领域压力。
- stats line：卡牌预览中的角色数值展示。

卡牌不是直接读取一句文案决定强度，而是通过行动模板、角色 profile、资源状态、基础数值评级和 scalingProfile 共同计算。

最新版本中，资源 actor 已保留 characterCardProfile，用于让卡牌模板读取角色原始基础数值和术式信息。

4. EX- 档位已加入卡牌模板数值体系

已在卡牌模板中加入 EX- 档位：

- rankMultipliers["EX-"] = 4.25
- efficiencyCostMultipliers["EX-"] = 0.45
- rankScores["EX-"] = 10.2

用途：给 SSS 与 EX 之间留出更细的强度层级，避免所有高位角色直接跳到 EX。

5. “咒力最大输出”机制已加入

已新增 ceMaxOutput / ceMaxOutputScore 作为卡牌模板内的派生数值。

当前计算思路：

ceMaxOutputScore =
- 咒力量评分 * 0.45
- 咒力操控评分 * 0.25
- 咒力效率评分 * 0.10
- 术式威力评分 * 0.20
- 咒术轴超出咒力量部分的补正
- 特殊关键词补正

特殊关键词补正目前识别：

- 石流
- Granite Blast
- 最大出力
- 最大输出
- 純愛砲 / 纯爱炮
- 茈
- 虚式
- World Slash
- 世界斩
- 六眼
- 无下限 / Limitless / 五条 / Gojo

当前设计结果：

- 六眼本身不会无条件把角色推到 EX 最大输出。
- 六眼会提供轻量补正。
- 六眼 + 无下限 / 五条语境会更容易达到 EX。
- 石流、纯爱炮、虚式茈、世界斩等明确“输出型术式”会获得更明显的最大输出补正。

6. “咒力最大输出”已接入卡牌伤害与领域压力

已完成接入：

- technique 类型卡牌：
  同时参考 ceOutput、ceMaxOutput、ceControl、techniquePower。

- ce_burst 类型卡牌：
  更强依赖 ceMaxOutput，适合作为“最大输出爆发”类卡牌。

- domainPressure：
  在领域压力中加入 ceMaxOutput 的部分影响。

7. 六眼专项审阅已开始并完成第一轮路径拆分

已确认六眼目前存在三条影响路径：

- 人生/构筑侧：computeAbilityBuildBonus 中六眼提供 abilityBuildBonus。
- 机制卡侧：sixEyes 机制锁定 controlScore / efficiencyScore 最低 SSS / 8，并给即时战力轴补正。
- 卡牌对战侧：通过 control、efficiency、resource、stability、ceCost、technique scaling、ceMaxOutput 等间接影响卡牌。

当前结论：

- 六眼不是“所有卡牌直接加固定伤害”的天赋。
- 六眼更偏向降低咒力浪费、提高操控、提高术式兑现率、改善领域友好度。
- 在卡牌中，六眼会表现为更低消耗、更高稳定性、更强术式效率。
- 若搭配无下限、五条、虚式等关键词，会进一步强化最大输出相关卡牌。
